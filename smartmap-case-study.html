Smart Map

UAE government spatial platform serving 2M+ residents. I redesigned the layer architecture and built a dual-mode interface — one for public exploration, one for government administrators.

UI UX Designer 2025–2026 Team Size: 5

Problem

  • Residents had no way to find nearest healthcare or education facilities with real-time availability
  • 40+ spatial datasets presented as flat layers — no context, no proximity search
  • No natural language search — users had to know exact dataset names to find anything
  • Government officers had no admin tools to manage and publish updated facility data

Solution

Built a spatial platform that helps UAE residents find what they need — nearest hospitals with available emergency beds, schools with open enrollment, clinics within walking distance — all through natural language search and radius-based exploration.

NL Search

"Find hospitals near me with emergency beds" — plain language, not dataset codes

Radius Search

Set a radius (1km, 5km, 10km) to find facilities within walking or driving distance

Real-Time Data

Live availability — emergency bed counts, school capacity, clinic hours — updated continuously

Admin Dashboard

Government officers publish and manage facility data, access controls, and layer visibility

Core Experience

Finding Nearest Facilities

Residents search in natural language — "hospitals near Al Majaz with available beds" — and the map immediately highlights matching facilities with distance, availability, and contact info.

Real-Time Healthcare Data

Each facility card shows live data: emergency bed availability, current wait times, operating hours. Healthcare workers update this through the admin dashboard — residents see it instantly on the map.

Education & Services

Beyond healthcare — the platform covers education (schools, enrollment status, ratings), government services, and infrastructure. Each category shows relevant real-time attributes specific to that service type.

Key Decisions

Dual-mode architecture

Separated public and admin interfaces completely. Tested a toggle approach — users couldn't find their way back. Two modes, zero confusion.

Category-first navigation

Replaced flat layer lists with semantic grouping. Users find data by intent ("I want to see schools"), not by technical dataset names.

Visible map state

Added persistent layer indicators so users always know what the map is currently showing. Eliminated the "what am I looking at?" problem.

No data pipeline changes

Designed a presentation layer on top of the existing GIS backend. Didn't require any data restructuring — reduced implementation risk.
