What is SmartMap?

SmartMap is a public-facing mapping platform that helps users in the UAE:

  • Find nearby services such as schools and hospitals.
  • Explore locations through seamless map-based interactions.
  • Access spatial data in a simple and accessible way.

It acts as a digital bridge, serving both the general public seeking everyday information and professionals working deeply with location-based data infrastructure.

Objective & Problems

The Objective

The core objective of this project was to evaluate the usability of the existing platform, identify friction points in navigation and service discovery, and ultimately improve structural clarity and usability for a massively wide audience.

Key UX Problems

  • Navigation overload: Too many unprioritized controls exposed simultaneously.
  • Layer complexity: Data overlaps causing visual noise and confusion.
  • Poor discoverability: Everyday users struggled to find basic public services.
  • High cognitive load: The interface catered exclusively to expert analysts, alienating general users.

User Types

1. General Users

Goal: Find nearby services quickly.

Pain: Complex interface.

2. Analysts / Planners

Goal: Explore layered spatial data.

Pain: Data overlap confusion.

3. Operators

Goal: Manage map layers.

Pain: Lack of structure.

UX Decisions

Architecting the Solution

Every design decision was focused on resolving friction. We utilized a strict Problem → Decision → Impact framework:

1. Navigation Complexity

→ Decision: Introduced a structured, unified toolbar.
→ Impact: Noticeably reduced user confusion during basic tasks.

2. Layer Management Overload

→ Decision: Grouped layers using a strict visual hierarchy.
→ Impact: Improved clarity and map legibility for Planners.

3. Service Discovery Friction

→ Decision: Added prominent category-based search chips (e.g., Schools, Hospitals).
→ Impact: Drastically faster query results for General Users.

4. High Cognitive Load

→ Decision: Applied aggressive progressive disclosure principles.
→ Impact: A vastly cleaner, less intimidating map experience.

5. Steep First-Time Use Curve

→ Decision: Added lightweight onboarding cues and contextual tooltips.
→ Impact: An immediately easier learning curve for non-technical residents.

Reflection

This project was a masterclass in learning how to aggressively simplify incredibly complex, data-dense public systems. The primary challenge was successfully balancing raw usability with data richness.

By forcing ourselves to design for both the everyday general user (who just needs a hospital) and the expert planner (who needs localized spatial analytics), we constructed a platform that scales smoothly across both ends of the technical spectrum without alienating either.
