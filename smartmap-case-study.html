Context & Objective

The Platform

SmartMap is utilized natively for viewing intense geospatial data, managing layer-based mapping infrastructures, and extracting high-level environmental insights.

UX Analysis Objective

The core objective of this project was to analyze the usability of the existing system, scientifically identify UX friction points, and structurally suggest actionable improvements for platform clarity and workflow efficiency.

User Types

Analyst

Behavior: Explores raw data.

Struggles with unstructured overlays.

Decision Maker

Behavior: Seeks quick insights.

Struggles with visual clarity.

Operator

Behavior: Manages active layers.

Struggles with system navigation.

Discoverability Issues

Critical functions and contextual insights are consistently buried under multiple nested menus, forcing users to memorize click paths rather than relying on intuitive progressive disclosure maps.

Opportunities & Solutions

Core Opportunity Areas

  • Simplify layer grouping and hierarchy.
  • Improve structural prioritization in the UI.
  • Reduce task steps for advanced querying.
  • Add explicit contextual guidance during map interactions.

Implementation Focus

Layer overload
→ Solution: Introduced a grouped, logical layer syntax system.
→ Impact: Massively reduced cognitive load.

Confusing spatial filters
→ Solution: Contextual filtering UI attached to the active map state.
→ Impact: Noticeably faster task completion and confidence.
