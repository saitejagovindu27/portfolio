Smart Map

UAE government spatial platform serving 2M+ residents. I redesigned the layer architecture and built a dual-mode interface — one for public exploration, one for government administrators.

Lead UX Designer Sharjah Government 2024–2025 GIS System

Problem

  • 40+ spatial datasets presented as flat layers — users couldn't find relevant data
  • No distinction between public residents and government administrators
  • Map state was invisible — users had no idea what layers were active
  • Search returned raw dataset names instead of meaningful results

Solution

Restructured the platform around two core ideas: intent-based navigation (users browse by what they want to do, not by dataset names) and dual-mode architecture (public discovery mode vs. professional admin mode).

Discovery Mode

Category-based entry for residents — browse by topic (transport, environment, zoning)

Professional Mode

Full layer controls, attribute tables, and export tools for GIS officers

Smart Search

Natural language queries — "show me schools near Al Majaz" instead of dataset codes

Layer Architecture

9 flat layers → 3 semantic groups with visual state indicators

Core Experience

Data Exploration

Users browse spatial data through category cards (Environment, Infrastructure, Demographics) instead of a flat layer list. Each category shows preview thumbnails and descriptions before loading onto the map.

Map Interaction

Active layers are visible through a persistent sidebar with toggle states. Clicking any feature opens a detail panel with attributes, related datasets, and export options — no more hunting through menus.

Admin Governance

Government officers manage layer permissions, publish datasets, and configure access controls through a dedicated admin panel — completely separated from the public interface.

Key Decisions

Dual-mode architecture

Separated public and admin interfaces completely. Tested a toggle approach — users couldn't find their way back. Two modes, zero confusion.

Category-first navigation

Replaced flat layer lists with semantic grouping. Users find data by intent ("I want to see schools"), not by technical dataset names.

Visible map state

Added persistent layer indicators so users always know what the map is currently showing. Eliminated the "what am I looking at?" problem.

No data pipeline changes

Designed a presentation layer on top of the existing GIS backend. Didn't require any data restructuring — reduced implementation risk.
