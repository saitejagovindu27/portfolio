SmartMap is a UAE government spatial platform used by two very different users:

  • Residents finding services like hospitals
  • Administrators managing spatial data

Both rely on the same interface, creating a challenge of balancing simplicity with complexity.

The problem

The platform had a layer system that made sense to the engineers who built it and no one else. Nine ungrouped categories, all visible at once, with toggle states that looked identical whether active or not. Residents had no way to browse by intent. The live bed availability data GÇö arguably the most valuable thing on the platform GÇö was buried so deep that none of the 4 users in our research sessions found it without being told it existed.

The harder problem was that government administrators needed the opposite experience GÇö full layer control, specialist data access, precision toggles. One interface. Two completely different mental models. That was the actual design challenge.

Design Direction

  • Simplify navigation into key actions
  • Introduce category-based discovery
  • Reduce map clutter using progressive disclosure
  • Improve onboarding and feedback

What we changed

We tore down the nine flat layers and restructured them into three grouped categories with distinctly visible toggle states. We bypassed the layer panel entirely for general users, adding a category filter directly over the map.

Map Interface

The default map state is now entirely clean. We utilized progressive disclosure to ensure services only populate when a user actively selects an intent category, reducing cognitive load.

Layer Controls

The restructured layer panel groups nine flat categories into three semantic sets. Users can now instantly distinguish active layers from inactive ones without reading every label.

Search & Discovery

Bypassing the standard layer panel with a dedicated category filter overlay. This makes critical live data, such as hospital bed availability, discoverable in a single click.

What I took from this

I went in thinking the problem was visual clutter. The research showed the real failure was invisible system state GÇö users had no idea what the map was currently showing them. That's a different problem entirely, and it changes what you design.

We locked the layer architecture before we properly tested with administrator users. Their needs were different enough that we made compromises I'd avoid if I ran it again GÇö with earlier, more deliberate testing across both user types from week one. That's the version I'd build next time.
