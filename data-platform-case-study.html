AI Data Platform

IST's flagship AI-powered data intelligence suite. I designed the complete product — from the marketing landing page to 8 integrated application modules — creating a unified experience for data cataloguing, spatial visualization, analytics, and governance.

UI UX Designer 2025 Team Size: 3

Problem

  • 8 modules built by different engineering pods — each with its own UI patterns and navigation
  • Users had to relearn the interface every time they switched modules
  • No shared design system — buttons, tables, and cards looked different everywhere
  • The landing page promised "AI-Powered Insights" but the application felt like 8 separate tools

Solution

Created a unified design language across all 8 modules and organized the platform around the data lifecycle — Ingest → Explore → Analyze → Govern — instead of the engineering architecture.

Core Experience

Data Exploration

Users discover datasets through smart search and category browsing. Each dataset shows metadata, preview, and lineage before loading — reducing wasted clicks on irrelevant data.

Spatial Analysis

Map Viewer integrates with catalogues — select a dataset, it loads on the map. Drawing tools, measurement, and attribute queries work across all spatial data without switching contexts.

Governance

Admins control access at the dataset, field, and user-group level. Audit logs track every data access event. Compliance reports are auto-generated.

AI Integration

AI powers three experiences: smart search across catalogues (natural language queries), automated data quality scoring on ingestion, and predictive analytics in the reports module. The AI is invisible — users get better results without having to "use AI" as a feature.

Key Decisions

Pipeline-based navigation

Organized by data lifecycle (Ingest → Explore → Analyze → Govern) instead of 8 sidebar items. Users think in tasks, not modules.

Unified design system

One component library across all modules. Users learn the interface once — buttons, tables, and interactions work the same everywhere.

Modular architecture

Each module can be enabled/disabled per client. The platform feels complete even when not all modules are deployed.

AI as infrastructure

AI isn't a tab — it's embedded in search, quality, and analytics. Users experience better results without cognitive overhead.
